# 편지 보내기 프로그램 설치 설명서

PC에서 편지를 작성한 다음 휴대폰을 통해 SMS로 발송하는 프로그램의 설치 가이드입니다.
처음 한 번만 따라하면 그 다음부터는 그냥 프로그램만 실행하면 됩니다.

---

## 준비물

- **PC**: Windows 10 또는 11
- **휴대폰**: Android (Samsung Galaxy 등)
- **PC와 휴대폰이 같은 Wi-Fi에 연결**되어 있을 것
- **PC의 .NET 8 Desktop Runtime** (없으면 1단계에서 설치)

---

## 1단계 - PC: .NET 8 Desktop Runtime 설치

편지 보내기 프로그램을 실행하는 데 필요합니다.

1. https://dotnet.microsoft.com/ko-kr/download/dotnet/8.0 접속
2. 화면 오른쪽 **".NET Desktop Runtime"** 항목에서 **x64** 옆 다운로드 버튼 클릭
3. 받은 설치 파일 더블클릭 → "설치" 버튼 → 끝

> 이미 다른 .NET 앱이 설치돼 있으면 이미 깔려있을 수 있습니다.
> 일단 다음 단계로 가서 프로그램이 정상 실행되면 이 단계는 건너뛰어도 됩니다.

---

## 2단계 - PC: KDE Connect 설치

PC와 휴대폰을 연결하는 무료 프로그램입니다.

**가장 쉬운 방법 (winget 사용):**

1. **시작 메뉴** → "PowerShell" 검색 → 실행
2. 아래 명령어를 그대로 복사해서 붙여넣고 Enter
   ```
   winget install KDE.KDEConnect
   ```
3. 설치가 끝날 때까지 기다림 (1~2분)

**winget이 없다면 직접 다운로드:**

1. https://kdeconnect.kde.org/download.html 접속
2. **Windows** 섹션의 다운로드 버튼 → 받은 .msi 파일 더블클릭
3. "다음 → 다음 → 설치" 진행

설치가 끝나면 자동으로 KDE Connect가 백그라운드에서 실행되며, **시스템 트레이(시계 옆)** 에 KDE Connect 아이콘이 생깁니다.

### 2-1. PC 부팅 시 자동 실행 확인

KDE Connect 인스톨러는 보통 자동 시작에 등록해주지만, 한 번 확인해두면 안전합니다.

1. **시작 메뉴** → **"설정"** → **앱** → **시작 프로그램**
2. 목록에서 **"KDE Connect"** 또는 **"KDE Connect Indicator"** 항목을 찾아 **"켜짐"** 으로 토글
3. 만약 목록에 없다면:
   - **Win + R** → `shell:startup` 입력 → Enter
   - 열린 폴더에 `C:\Program Files\KDE Connect\bin\kdeconnect-indicator.exe` 의 바로가기를 복사 (마우스 오른쪽 → 보내기 → 바탕 화면에 바로 가기 만들기 → 만든 바로가기를 startup 폴더로 이동)

이렇게 해두면 **PC 부팅 후 자동으로 KDE Connect가 실행**되어 트레이에 떠 있습니다.

---

## 3단계 - 휴대폰: KDE Connect 앱 설치

1. 휴대폰의 **Play 스토어** 앱 실행
2. 검색창에 **"KDE Connect"** 입력
3. KDE Connect 앱 (KDE 커뮤니티 제작) 설치
4. 설치 후 한 번 실행

---

## 4단계 - PC와 휴대폰 페어링

**여기부터는 PC와 휴대폰이 같은 Wi-Fi에 있어야 합니다.**

1. 휴대폰의 KDE Connect 앱 열기
2. 메인 화면에 자동으로 PC 이름이 보임 (예: "DESKTOP-XXXXX")
3. PC 이름 탭 → **"페어링 요청"** 버튼 누르기
4. **PC 화면 오른쪽 아래** 에 페어링 요청 알림이 뜸 → **"수락"** 클릭
5. 양쪽에서 "페어링됨" 상태가 되면 완료

> 휴대폰에 PC가 안 보이면:
> - PC와 휴대폰이 **같은 Wi-Fi**에 있는지 다시 확인
> - PC의 KDE Connect 트레이 아이콘이 살아있는지 확인 (없으면 시작 메뉴에서 KDE Connect 다시 실행)
> - Windows 방화벽 알림이 떴다면 "허용"

---

## 5단계 - 휴대폰: SMS 권한 부여 (가장 중요!)

이 단계를 빠뜨리면 PC에서 발송해도 문자가 안 갑니다.

### 5-1. KDE Connect 앱에서 SMS 플러그인 켜기

1. 휴대폰의 **KDE Connect 앱** 열기
2. 페어링된 PC 이름 탭
3. **"플러그인 설정"** (또는 우측 상단 톱니바퀴 아이콘)
4. 목록에서 **"SMS Messages"** 또는 **"SMS 메시지"** 항목 → **체크**

### 5-2. 안드로이드 시스템 권한 모두 허용

1. 휴대폰의 **설정** 앱 열기 (KDE Connect 앱 안 말고, 폰의 시스템 설정)
2. **앱** → 목록에서 **KDE Connect** 탭
3. **권한** 탭
4. 목록에 있는 모든 권한 항목 (특히 **SMS, 전화, 연락처**) → 들어가서 **"허용"** 으로 체크

> SMS 권한 한 번도 안 줬으면 보내기 시도 시 폰에서 silent하게 무시당합니다.
> 모든 권한을 허용해두는 것이 가장 확실합니다.

---

## 6단계 - 휴대폰: KDE Connect가 항상 살아있게 만들기

이 작업을 안 하면 폰이 절전 모드로 들어갈 때 KDE Connect가 잠들어버려서,
PC에서 보낸 문자가 도착하지 않거나 한참 늦게 도착합니다.

**Galaxy (One UI) 기준:**

### 6-1. 배터리 최적화 제외

1. 휴대폰의 **설정** → **배터리 및 디바이스 케어** → **배터리** → **백그라운드 사용 한도**
2. **"절대 잠자기 앱(자동 절전 예외 앱)"** 항목 → **추가** 버튼 → 목록에서 **KDE Connect 선택**
3. 만약 **"잠자는 앱"** 또는 **"심층 잠자는 앱"** 목록에 KDE Connect가 있다면 **제거**

### 6-2. 앱별 배터리/네트워크 설정

1. 설정 → **앱** → **KDE Connect** → **배터리** → **"제한 없음"** 선택
2. 설정 → **앱** → **KDE Connect** → **모바일 데이터** → **"백그라운드 데이터 사용 허용"** 켜기

### 6-3. (참고) 자동 실행에 대해

최근 삼성 폰(One UI 5 이상)은 앱별 "자동 실행" 토글이 따로 없습니다.
6-1의 **"절대 잠자기 앱"에 KDE Connect 추가**가 사실상 자동 실행 + 항상 실행을 모두 보장합니다.

만약 폰이 구형 One UI거나 다른 제조사라서 **"자동 실행 앱"** / **"Auto-start"** 메뉴가 보이면, 거기서도 KDE Connect를 켜주세요.

### 6-4. 동작 확인 (꼭 한 번)

설정이 끝나면 폰을 한 번 재부팅 → 1~2분 후 휴대폰의 알림 영역(상단 끌어내리기)에 **"KDE Connect가 실행 중"** 같은 영구 알림이 떠있는지 확인.

- 알림이 떠있다 → 자동 실행 + 항상 실행 정상 동작
- 알림이 안 보인다 → KDE Connect 앱을 한 번 열어주기 (그래도 안 살아있으면 6-1, 6-2 다시 점검)

> 다른 안드로이드 기기는 메뉴 위치가 조금 다를 수 있지만,
> 핵심은 "**KDE Connect를 배터리 절약·자동 종료 대상에서 빼기**" 입니다.

## 7단계 - PC: 편지 보내기 프로그램 첫 실행 및 설정

1. **편지 보내기.exe** (또는 `h_sms_sender.exe`) 더블클릭
2. 프로그램 창이 뜨면 상단 메뉴 → **[설정]** → **[설정 열기...]**

### 7-1. 휴대폰 선택

1. "연결할 휴대폰" 옆 **[새로고침]** 버튼 클릭
2. 페어링된 휴대폰 이름이 콤보박스에 뜸 (예: "Galaxy S10e")
3. 그것을 선택

> 콤보박스에 "(검색된 기기 없음)"으로 뜨면:
> - PC의 KDE Connect가 실행 중인지 확인 (시스템 트레이 아이콘)
> - 4단계 페어링이 정상적으로 끝났는지 다시 확인

### 7-2. 폰트 설정 (선택)

1. "편집창 폰트" 옆 **[폰트 변경...]** → 원하는 폰트와 크기 선택 → 확인
2. 미리보기로 적용된 모습 확인

### 7-3. 받는 사람 등록

1. 하단 "받는 사람 목록" 표에 한 줄씩 입력
   - **이름** 칸: 표시할 이름 (예: "엄마")
   - **전화번호** 칸: 휴대폰 번호 (예: `01012345678`, 하이픈 없이)
2. 여러 명을 미리 등록해두면 발송할 때 체크박스로 선택

### 7-4. 저장

화면 오른쪽 아래 **[확인]** 버튼 → 저장 후 설정창 닫힘

---

## 8단계 - 사용 방법

1. 메인 화면 **왼쪽** 에서 보낼 사람을 체크 (한 명 이상)
2. **오른쪽 큰 편집창** 에 편지 내용 작성
3. 화면 오른쪽 아래 **[발송]** 버튼 클릭
4. 확인 다이얼로그 → **[예]**
5. "발송 완료" 메시지 → 휴대폰에 SMS 도착 확인

새 편지를 쓰고 싶으면 메뉴 → **[파일] → [새 편지]** (또는 Ctrl+N)

이전에 보낸 편지를 찾고 싶으면 메뉴 → **[기록] → [발송 기록 보기...]** (또는 Ctrl+H)

## 문제 해결

### "검색된 기기 없음" 으로 표시될 때
- PC의 KDE Connect 앱이 실행 중인지 확인 (시스템 트레이 아이콘)
- PC와 휴대폰이 같은 Wi-Fi에 있는지 확인
- 페어링이 살아있는지 확인 (휴대폰의 KDE Connect 앱에서 PC 이름 옆에 "연결됨" 표시)

### "전송 완료"라고 뜨는데 휴대폰에 안 올 때
- **5단계 SMS 권한**이 부여되어 있는지 다시 확인
- 휴대폰 KDE Connect가 잠들어 있을 가능성 → **6단계 배터리 최적화** 설정 다시 확인
- 휴대폰의 KDE Connect 앱을 한 번 강제로 열어서 "깨운" 다음 다시 발송

### "kdeconnect-cli.exe를 찾을 수 없습니다"
- KDE Connect를 설치한 적이 없거나 지운 경우 → 2단계부터 다시
- 비표준 위치에 설치한 경우 → 설정창에서 "KDE Connect CLI 경로" 옆 **[찾아보기]** 로 직접 지정 (보통 `C:\Program Files\KDE Connect\bin\kdeconnect-cli.exe`)

### 발송이 너무 늦게 도착함
- 6단계 배터리 최적화 설정이 제대로 안 됐을 가능성
- 휴대폰을 잠시 깨운 다음(화면 한 번 켰다 끄기) 발송하면 즉시 도착

---

## 설정 파일 위치 (참고)

- 설정: `%APPDATA%\h_sms_sender\config.json`
- 발송 기록: `%APPDATA%\h_sms_sender\history.json`

문제 발생 시 위 파일들을 한 번 지우고 프로그램을 재실행하면 초기 상태로 돌아갑니다.
(받는 사람 목록도 같이 지워지니 주의)
